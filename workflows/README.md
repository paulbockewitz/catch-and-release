# workflows/

Markdown SOPs that tell the agent exactly what to do — step by step.

## What a good workflow contains

1. **Objective** — one sentence on what this accomplishes
2. **Inputs** — what you need before starting (data, credentials, IDs)
3. **Steps** — ordered list of actions; reference `tools/` scripts by name
4. **Outputs** — what gets produced and where it lands
5. **Edge cases** — known failure modes and how to handle them
